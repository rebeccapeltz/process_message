1
What makes this "agentic"?
▾
A regular API call asks Claude a question and gets back text. An agentic loop is different: Claude is given tools (functions it can call), and it decides — based on your natural language — which tools to use, in what order, with what parameters.
The AI orchestrates; your code does the work. Claude never directly touches your database or draws the chart — it just decides what to call.
2
How tool use (function calling) works
▾
You define tools in your API request as JSON schemas. For example:

get_data(metric, period) — fetches a dataset
render_chart(type, labels, values, title) — specifies a chart

Claude reads your message and the tool definitions, then responds with a tool_use block instead of plain text — telling your code exactly which function to call and what parameters to pass. Your code executes it, sends the result back, and Claude continues reasoning.
This is the key difference from the simple demo: Claude is no longer just answering — it's deciding.
3
Why this answers the classroom objection
▾
The student's question was smart: "Why would I want AI responsible for fetching data AND creating the chart?" The answer: you don't. The agent pattern separates concerns clearly:

• AI's job: understand intent, pick tools, pick parameters
• Your code's job: fetch data reliably, render charts correctly

Claude is the decision-maker, not the executor. You maintain full control over what your code actually does.
4
The two-call pattern in this demo
▾
Call 1 — Intent + Tool Selection: Send user query + tool definitions. Claude responds with a tool_use block: which tool to call, with what params. This is where keyword extraction happens — Claude maps "revenue" → metric="revenue", "bar chart" → chart_type="bar".

Call 2 — Render Decision: After fetching real data, send it back to Claude and ask it to specify the chart mapping. Claude returns structured JSON your Chart.js code can use directly.
In production agents, this loop can have many more steps — Claude keeps calling tools until it has everything it needs.
5
Using your own data endpoint
▾
Replace the URL in Step 2 with any public JSON endpoint. The agent works best when your JSON has consistent field names (e.g., month, revenue, units). CSV endpoints also work — the demo converts them automatically.

Example structure:
[{"month":"Jan","revenue":42000,"units":310}, ...]
Claude reads the field names from your actual data to decide how to map them to chart axes — no hardcoding required.
